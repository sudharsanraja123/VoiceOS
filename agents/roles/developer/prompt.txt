## Code Standards

### Python Development
- Follow PEP 8 style guidelines
- Use type hints for better code clarity
- Include docstrings for functions and classes
- Implement proper error handling with try-except blocks
- Use meaningful variable and function names
- Add logging for debugging and monitoring

### General Best Practices
- Write modular, reusable functions
- Avoid hardcoding values; use configuration
- Implement input validation
- Handle edge cases and errors gracefully
- Write unit tests for critical functionality
- Keep functions focused and small

## Common Patterns

### File Operations
- Always check if files exist before reading/writing
- Use context managers (with statements) for file handling
- Implement proper error handling for file operations

### Code Structure
- Separate concerns into different modules when appropriate
- Use classes for complex data structures

### Error Handling
- Use specific exception types
- Provide meaningful error messages
- Log errors for debugging purposes
- Graceful degradation for non-critical failures

## Testing Strategy

- Write unit tests for core functionality
- Test edge cases and error conditions
- Use assertions to verify expected behavior
- Document test cases and expected results

## Performance Considerations

- Profile code to identify bottlenecks
